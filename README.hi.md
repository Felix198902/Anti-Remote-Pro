## 🔥 Anti-Remote-Pro क्यों

### 🔒 1. पूर्ण गोपनीयता — आपका कोड लैपटॉप से बाहर नहीं जाता

- **100% लोकल पाइपलाइन।** सभी बटन डिटेक्शन, DOM स्कैनिंग, कमांड फ़िल्टरिंग आपकी VS Code / Antigravity प्रक्रिया के अंदर चलती है। कोई प्रॉम्प्ट, स्क्रीनशॉट, या diff कभी मशीन से बाहर नहीं जाता।
- **केवल localhost डिबग पोर्ट।** CDP `127.0.0.1:9333` पर सख्ती से बाँधा गया है। LAN, कैफ़े Wi-Fi, या को-वर्किंग नेटवर्क में कोई बाहरी होस्ट इस तक नहीं पहुँच सकता — सॉकेट बाइंडिंग से सत्यापित।
- **शून्य टेलीमेट्री।** ऑटो-स्वीकृत कोर कोई एनालिटिक्स, क्रैश रिपोर्ट, या "अनाम उपयोग आँकड़े" नहीं भेजता। एकमात्र आउटबाउंड ट्रैफ़िक Telegram (केवल यदि जोड़ें) और लाइसेंस सर्वर (केवल कुंजी सत्यापन) है।
- **केवल व्हाइटलिस्ट टारगेट।** CDP **केवल** Agent पैनल से मेल खाने वाले `vscode-webview://` टारगेट से जुड़ता है। आपके अन्य टैब, ब्राउज़र सब-एजेंट्स, या non-Antigravity Chromium टारगेट्स से जुड़ने से इनकार करता है।
- **हस्ताक्षरित कोर पेलोड।** Pro कोर इंजन Ed25519-हस्ताक्षरित पेलोड के रूप में आता है, Node worker में सत्यापित होता है, RAM में संकलित — डिस्क पर नहीं लिखा जाता, MITM द्वारा छेड़खानी नहीं।
- **Antigravity की कोई फ़ाइल संशोधित नहीं।** हम बाइनरी को पैच, रीपैक, या हुक नहीं करते। शुद्ध बाहरी CDP — अनइंस्टॉल करें और शून्य निशान।

### 📱 2. Telegram रिमोट कंट्रोल — कहीं से भी Antigravity चलाएँ

अपनी IDE को Telegram बॉट से एक बार जोड़ें, और आपका फ़ोन Agent के लिए पूर्ण रिमोट बन जाता है:

- **`/prompt <टेक्स्ट>`** — ग्रह पर कहीं से भी सक्रिय बातचीत में नया प्रॉम्प्ट भेजें। Telegram के 4096-वर्ण विभाजित संदेशों को स्वतः जोड़ता है।
- **`/peek [N]`** — Telegram में ही Agent का नवीनतम जवाब पढ़ें, लैपटॉप जगाने की ज़रूरत नहीं।
- **कैप्शन के साथ फ़ोटो भेजें** — स्क्रीनशॉट को प्रॉम्प्ट से जोड़ता है, विज़ुअल डिबगिंग के लिए।
- **`/pause` और `/resume`** — फ़ोन से ग्लोबल किल-स्विच पलटें। क्रॉस-विंडो फ़ाइल लॉक गारंटी देता है कि हर Antigravity विंडो तुरंत आज्ञा माने।
- **`/stop`** — वर्तमान में जनरेट हो रहे जवाब को रोकें।
- **HMAC-हस्ताक्षरित कमांड।** हर Telegram कमांड डिवाइस-प्रति HMAC कुंजी से सत्यापित होता है। लीक हुआ Telegram चैट आपकी IDE को नियंत्रित करने के लिए पर्याप्त नहीं है।
- **एकल-मास्टर चुनाव।** 5 Antigravity विंडो खुली हों तो केवल एक मास्टर लॉक जीतती है और कमांड्स प्रोसेस करती है — कोई डुप्लिकेट, कोई रेस कंडीशन नहीं।

### 🛡️ 3. उद्योग-प्रथम बैन-विरोधी रणनीति

अधिकांश ऑटोक्लिकर खातों को फ़्लैग करवाते हैं क्योंकि वे एंडपॉइंट्स को निश्चित कैडेंस पर पीटते हैं और तब क्लिक करते हैं जब इंसान अभी भी टाइप कर रहा होता है। Anti-Remote-Pro बिल्कुल विपरीत इंजीनियर्ड है:

- **मानवीय लय।** 500 ms सक्रिय पोलिंग + **±20% रैंडम जिटर** + लगातार त्रुटियों पर एक्सपोनेंशियल बैकऑफ़ (30s पर सीमित)। फ़िंगरप्रिंट के लिए कोई मशीन-परिपूर्ण अंतराल नहीं।
- **निष्क्रिय स्मार्ट-स्लीप।** बिना सक्रिय CDP सत्र, इंजन 5s हार्टबीट पर गिरता है — जब आप दूर हों तो व्यावहारिक रूप से शून्य IPC दबाव।
- **20-सेकंड उपयोगकर्ता-सक्रिय छूट विंडो।** यदि उपयोगकर्ता चैट में टाइप कर रहा, webview स्क्रॉल कर रहा, या कर्सर हिला रहा है, **सभी ऑटो-क्लिक टाल दिए जाते हैं**। बॉट आपके रुकने का इंतज़ार करता है, कभी पढ़ते समय साथ क्लिक नहीं करता।
- **कोई बाइनरी पैचिंग नहीं।** Antigravity फ़ाइलें, settings.json, डेटा डायरेक्टरी इस एक्सटेंशन के लिए केवल पढ़ने योग्य हैं। ऑटो-अपडेट्स और अखंडता जाँच साफ़ पास होती हैं।
- **केवल लूपबैक CDP।** कोई TCP listener नहीं, कोई रिमोट फ़ॉरवर्डिंग नहीं। `--remote-debugging-port=9333` Microsoft, Google, JetBrains द्वारा आंतरिक रूप से उपयोग किया गया वही सामान्य Electron डिबग फ़्लैग है।
- **कठोर कमांड फ़ायरवॉल।** बिल्ट-इन ब्लॉकलिस्ट + वैकल्पिक सख्त ऐलोलिस्ट + 50+ क्यूरेटेड **Safety Presets** (rm -rf, force-push, drop database, fork bombs, डिस्क वाइपर, रजिस्ट्री हटाने…)। Agent ज़ोर डाले तो भी बॉट विनाशकारी कमांड चलाने से *इनकार* करेगा।
- **विस्तार बटन के लिए प्रति-सत्र एक क्लिक।** "अनंत पुनः-खुलने" लूप को ख़त्म करता है जो दुरुपयोग ह्यूरिस्टिक्स को ट्रिगर कर सकता है।
- **आधिकारिक Browser सब-एजेंट को तुरंत रास्ता।** जब AG का अपना ब्राउज़र टूलिंग पोर्ट 9222 सक्रिय करता है, हमारा worker अलग हो जाता है — कोई पोर्ट टकराव नहीं।

> संयुक्त प्रभाव: Agent बैकएंड के दृष्टिकोण से आप औसत से थोड़ा तेज़, बेदाग़ विनम्र मानव उपयोगकर्ता दिखते हैं।

### 🔗 5. Google साइन-इन मल्टी-डिवाइस सिंक — एक खाता, सभी मशीनें

**Google से एक बार साइन इन करें** और Anti-Remote-Pro आपके साथ हर जगह जाता है — डेस्कटॉप, लैपटॉप, घर का PC, कार्य वर्कस्टेशन, Windows, macOS, Linux।

- **एक-क्लिक Google OAuth।** `Anti-Remote-Pro: Login with Google` ब्राउज़र खोलता है, मानक Google OAuth पूरा करता है, और सुरक्षित `vscode://Pro.antigravity-autoaccept/auth` URI handler के माध्यम से IDE पर लौटाता है। कोई लाइसेंस कुंजी कॉपी-पेस्ट नहीं, कोई ईमेल नृत्य नहीं।
- **Pro लाइसेंस डिवाइसों के बीच स्वतः सिंक।** सब्सक्रिप्शन Google खाते से बंधी है, मशीन से नहीं। डिवाइस #2, #3, #10 पर इंस्टॉल करें — साइन इन करें और Pro तुरंत अनलॉक। लाइसेंस सर्वर हर रिफ्रेश पर खाते के विरुद्ध हर `machineId` सत्यापित करता है।
- **असीमित डिवाइसों पर एक साथ चलाएँ।** कार्य लैपटॉप, घर का डेस्कटॉप, रिमोट dev बॉक्स, हर ग्राहक के लिए VM — सभी एक ही खाते के तहत समानांतर।
- **समानांतर बेड़े।** हर डिवाइस अपने CDP / Swarm / कमांड फ़ायरवॉल के साथ स्वतंत्र रूप से संचालित होती है। लैपटॉप बातचीत A चला सकता है जबकि डेस्कटॉप बातचीत B — कोई दूसरे से फ़ोकस नहीं चुराता।
- **प्रति-डिवाइस Telegram पेयरिंग।** हर मशीन Telegram बॉट से अलग जुड़ती है और अपनी HMAC कुंजी पाती है। पेयरिंग में नाम दें ("MacBook", "WorkPC") और एक ही Telegram चैट से व्यक्तिगत रूप से संबोधित करें।
- **एक ही मशीन पर क्रॉस-विंडो समन्वय।** फ़ाइल-लॉक प्रोटोकॉल पॉज़/रिज़्यूम स्थिति को एक सेकंड से कम में सिंक रखता है।
- **तत्काल निरस्तीकरण।** `Anti-Remote-Pro: Logout` लोकल टोकन मिटाता है और सर्वर को उस डिवाइस को अमान्य करने को कहता है।
- **Refresh Pro Status।** योजना बदली? एक कमांड नवीनतम अधिकार पुनः लाता है — कोई पुनर्स्थापना या पुनरारंभ नहीं।

> व्यावहारिक: 3 कंप्यूटरों पर एक-एक बार साइन इन करें, हर एक को एक ही Telegram बॉट से जोड़ें, और फ़ोन आपके पूरे कोडिंग बेड़े के लिए केंद्रीय रिमोट बन जाता है।

## ⚡ कोर क्षमताओं की झलक

- **ऑटो-क्लिक** Run / Accept / Always Allow / Allow this conversation / Continue / Retry — Agent पैनल पर, प्राथमिकता-जागरूक (`Run` हमेशा `Accept` से जीतता है)।
- **फ़ाइल संपादन स्वतः स्वीकार** एक टॉगल से — किसी भी समय बंद करके diff मैन्युअल समीक्षा करें।
- **कमांड फ़ायरवॉल** — पैटर्न ब्लॉक या सख्त ऐलोलिस्ट मोड, शब्द-सीमा मिलान के साथ।
- **Safety presets** — एक-क्लिक 50+ विनाशकारी कमांड पैटर्न आयात।
- **Swarm मोड (Pro)** — Agent Manager की पेंडिंग बातचीतों के बीच स्वतः घूमता है।
- **Telegram Remote (Pro)** — पूर्ण IDE रिमोट: `/prompt`, `/peek`, फ़ोटो प्रॉम्प्ट्स, `/pause`, `/resume`, `/stop`, HMAC ऑथ।
- **Google साइन-इन + मल्टी-डिवाइस सिंक (Pro)** — एक Google खाता, असीमित डिवाइस।
- **क्रॉस-विंडो पॉज़** — एक विंडो में पॉज़ <1s में फ़ाइल-लॉक के माध्यम से सभी विंडो को।
- **गतिविधि लॉग + डैशबोर्ड** — हर क्लिक, हर CDP सत्र, हर Telegram कमांड लाइव।
- **डायग्नोस्टिक डंप** — संरचित डिबग पेलोड क्लिपबोर्ड पर कॉपी।
- **ऑटो-फ़िक्स शॉर्टकट** — `--remote-debugging-port` के लिए Windows `.lnk` पैचर।
- **हार्टबीट सेल्फ-हीलिंग** — मृत MutationObservers हर 10s में स्वतः पुनः-इंजेक्ट।

## यह क्या करता है

जब Antigravity Agent फ़ाइल संपादन, टर्मिनल कमांड्स, या टूल अनुमतियाँ माँगता है, यह एक्सटेंशन उन्हें स्वतः स्वीकार करता है ताकि हर बटन मैन्युअल क्लिक न करना पड़े।

**दो रणनीतियाँ, शून्य हस्तक्षेप:**

| रणनीति | क्या संभालती है | कैसे |
|---|---|---|
| **VS Code Commands** (500 ms + जिटर) | Agent चरण, टर्मिनल कमांड्स | Antigravity के नेटिव accept कमांड्स को कॉल करता है |
| **CDP + MutationObserver** (इवेंट-संचालित) | Run, Accept, Always Allow, Continue, Retry | प्रति टारगेट एक बार स्क्रिप्ट इंजेक्ट, DOM परिवर्तनों पर तुरंत प्रतिक्रिया |

## मल्टी-Agent वर्कफ़्लो

### ⚠️ Agent Manager सीमा (Swarm मोड के बिना)

Antigravity का Agent Manager एक **एकल साझा webview** का उपयोग करता है — केवल सक्रिय बातचीत का DOM रेंडर होता है। पृष्ठभूमि बातचीतें पूरी तरह unmount होती हैं। Swarm मोड के बिना, VS Code Commands API और CDP दोनों केवल वर्तमान दृश्यमान बातचीत तक पहुँच सकते हैं।

### ✅ समाधान: Swarm मोड (Pro)

Swarm मोड सभी पेंडिंग बातचीतों के बीच फ़ोकस घुमाकर इसे हल करता है:

- साइडबार में हर बातचीत स्कैन करता है और पेंडिंग काम वाली चुनता है।
- आपकी idle-seconds सेटिंग का सम्मान करता है — टाइप करते समय फ़ोकस नहीं चुराता।
- क्रॉस-विंडो जागरूक: किसी एक विंडो को पॉज़ करने से सभी पॉज़ हो जाते हैं।
- डिफ़ॉल्ट रूप से सुरक्षित: हस्ताक्षरित कोर पेलोड, कमांड फ़ायरवॉल अभी भी लागू।

### फ़ॉलबैक: Workspace डुप्लिकेट (मुफ़्त)

Pro लाइसेंस नहीं है तो भी 2–3 एजेंट्स समानांतर चलाएँ:

1. **File → Duplicate Workspace** क्लिक करें
2. एक ही प्रोजेक्ट से जुड़ी दूसरी Antigravity विंडो खुलती है
3. विंडो 1 में एक चैट, विंडो 2 में दूसरी
4. हर विंडो का अपना webview — एक्सटेंशन **दोनों विंडो में एक साथ** बटन ऑटो-क्लिक करता है

## यह कैसे काम करता है

### स्थायी CDP + MutationObserver

एक्सटेंशन Chromium के DevTools Protocol के लिए एक **स्थायी ब्राउज़र-स्तरीय WebSocket** कनेक्शन रखता है। हर 1.5s पोल करने के बजाय, यह प्रति टारगेट एक बार **MutationObserver** पेलोड इंजेक्ट करता है। ऑब्ज़र्वर तुरंत प्रतिक्रिया करता है जब React नए बटन तत्व माउंट करता है, स्ट्रीमिंग आउटपुट के दौरान CPU स्पाइक्स रोकने के लिए 100ms लीडिंग-एज थ्रॉटल के साथ। एक्सटेंशन **केवल-व्हाइटलिस्ट टारगेट फ़िल्टर** का उपयोग करता है।

### सिंगल-पास DOM स्कैनर

बटन स्कैनर प्रति चक्र **DOM ट्री को ठीक एक बार** चलता है, हर नोड के विरुद्ध सभी कीवर्ड एक साथ जाँचता है। यह O(D) है O(N×D) के बजाय। प्राथमिकता-जागरूक मिलान सुनिश्चित करता है कि DOM क्रम की परवाह किए बिना `Run` हमेशा `Accept` को हराता है।

### Webview गार्ड

Antigravity का Agent पैनल एक पृथक Chromium प्रक्रिया (OOPIF) में चलता है। इंजेक्ट किया गया स्क्रिप्ट `scanAndClick()` के अंदर एक स्थगित `isAgentPanel()` जाँच का उपयोग करता है — हर स्कैन पर `.react-app-container` गतिशील रूप से सत्यापित करता है।

### हार्टबीट सेल्फ-हीलिंग

CDP कनेक्शन हर हार्टबीट चक्र (10s) में सत्रों को सत्यापित करता है। यदि किसी सत्र का MutationObserver मृत है, यह बिना पुनः-कनेक्शन के ऑब्ज़र्वर को स्वतः पुनः-इंजेक्ट करता है। लगातार 3 बार अप्राप्य सत्र साफ़ रूप से अलग किए जाते हैं।

### विस्तार बटन लूप रोकथाम

विस्तार-प्रकार के बटन एक **प्रति-सत्र एक बार क्लिक** नियम का उपयोग करते हैं: एक बार क्लिक पर, वे उस CDP सत्र के लिए स्थायी रूप से दबा दिए जाते हैं। यह अनंत ओवरले पुनः-खुलने वाले लूप को रोकता है।

### बटन पहचान

Agent पैनल के अंदर, एक `TreeWalker` गलत-सकारात्मक रोकने के लिए `startsWith` मिलान + **शब्द-सीमा जाँच** का उपयोग करके टेक्स्ट सामग्री द्वारा बटन खोजता है:

| प्राथमिकता | टेक्स्ट | मेल खाता है |
|---|---|---|
| 1 | `run` | "Run Alt+d" बटन |
| 2 | `accept` | Accept बटन |
| 3 | `always allow` | अनुमति प्रॉम्प्ट्स |
| 4 | `allow this conversation` | बातचीत-स्कोप अनुमतियाँ |
| 5 | `allow` | अनुमति प्रॉम्प्ट्स |
| 6 | `retry` | Retry प्रॉम्प्ट्स |
| 7 | `continue` | Agent इनवोकेशन सीमा फिर से शुरू |

### कमांड फ़िल्टरिंग

ब्लॉक और अनुमति कमांड सूचियाँ Run बटन के ऊपर के कोड ब्लॉक के विरुद्ध **शब्द-सीमा मिलान** का उपयोग करती हैं। उदाहरण: `rm` ब्लॉक करना `rm -rf /tmp` को ब्लॉक करेगा लेकिन `yarn format` या `npm run build` को **नहीं**।

### बॉट कुछ घंटों के बाद काम करना बंद कर देता है

**कारण:** या तो (a) Antigravity चुपचाप Electron प्रक्रिया को पुनरारंभ करता है और नई प्रक्रिया में `--remote-debugging-port=9333` नहीं है, या (b) webview का निष्पादन संदर्भ नेविगेशन / हॉट-रीलोड द्वारा साफ़ कर दिया गया है।

**फ़िक्स:** हार्टबीट सेल्फ-हीलिंग मृत ऑब्ज़र्वर्स को स्वतः पहचानता और पुनः-इंजेक्ट करता है। यदि काम न करे, **सभी** Antigravity विंडो बंद करें और पैच किए गए शॉर्टकट से फिर से खोलें।

### बॉट ON है लेकिन कुछ क्लिक नहीं कर रहा

1. **OFF → ON टॉगल करें** — स्टेटस बार आइकन दो बार क्लिक करें
2. **डिबग पोर्ट जाँचें** — `http://127.0.0.1:9333/json/list` ब्राउज़र में खोलें। यदि इनकार करे, तो पोर्ट मृत है
3. **Output लॉग्स जाँचें** — `Ctrl+Shift+U` → ड्रॉपडाउन → `Anti-Remote-Pro`। `[CDP] ✓ Thread` पंक्तियाँ खोजें

### स्टेटस बार आइकन इंस्टॉल के बाद नहीं दिखता

1. `Ctrl+Shift+P` → `Reload Window`
2. सत्यापित करें कि VSIX **निर्भरताओं के साथ** बनाया गया था (`ws` पैकेज शामिल होना चाहिए)

## सुरक्षा FAQ

**इसे `--remote-debugging-port` की आवश्यकता क्यों है?**

Antigravity का Agent पैनल एक पृथक Chromium प्रक्रिया में चलता है। VS Code Extension API उसके अंदर Run / Accept / Allow बटनों को देख या उनसे बातचीत नहीं कर सकता — वे React UI तत्व हैं जिनके पास कोई पंजीकृत कमांड नहीं है। एक localhost पोर्ट पर Chrome DevTools Protocol (CDP) उन तक पहुँचने का एकमात्र तरीक़ा है।

**क्या यह सुरक्षित है?**

- **केवल localhost** — पोर्ट `127.0.0.1` से बँधता है, `0.0.0.0` नहीं। कोई बाहरी मशीन कनेक्ट नहीं कर सकती।
- **ऑटो-स्वीकृत कोर में शून्य टेलीमेट्री** — DOM स्कैनिंग, बटन मिलान, कमांड फ़िल्टरिंग कभी नेटवर्क कॉल नहीं करती।
- **एकमात्र आउटबाउंड ट्रैफ़िक**: (a) लाइसेंस सत्यापन, (b) Telegram Bot API (केवल यदि जोड़ें), (c) Swarm मोड हस्ताक्षरित कोर पेलोड। तीनों डिफ़ॉल्ट रूप से बंद।
- **हस्ताक्षरित Pro पेलोड।** Swarm कोर इंजन Ed25519-हस्ताक्षरित है और RAM में संकलित होने से पहले Node worker में सत्यापित होता है।
- **मानक dev वर्कफ़्लो** — `--remote-debugging-port` VS Code एक्सटेंशन डेवलपर्स और Electron ऐप डिबगिंग द्वारा उपयोग किया गया वही फ़्लैग है।
- **शॉर्टकट पैचर सीमित है** — ऑटो-फ़िक्स केवल उन `.lnk` फ़ाइलों को संशोधित करता है जिनका टारगेट पथ "Antigravity" रखता है।
- **कोई Antigravity फ़ाइल संशोधित नहीं** — यह एक्सटेंशन पूरी तरह एक बाहरी VS Code एक्सटेंशन के रूप में चलता है।

**क्या मेरी Telegram चैट मेरा कोड पढ़ेगी?**

केवल वह जो *आप* स्पष्ट रूप से माँगते हैं। `/peek` सक्रिय Agent पैनल का दृश्यमान टेक्स्ट लौटाता है; `/prompt` आपका संदेश अंदर भेजता है। एक्सटेंशन कभी सक्रिय रूप से कोड, diffs, स्क्रीनशॉट, या टेलीमेट्री Telegram पर पुश नहीं करता।
