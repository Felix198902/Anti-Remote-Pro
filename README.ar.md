## 🔥 لماذا Anti-Remote-Pro

### 🔒 1. خصوصية مطلقة — كودك لا يغادر حاسوبك أبداً

- **خط أنابيب محلي 100%.** كل اكتشاف الأزرار ومسح DOM وتصفية الأوامر يعمل داخل عملية VS Code / Antigravity الخاصة بك. لا يغادر أي prompt أو لقطة شاشة أو diff الجهاز أبداً.
- **منفذ التصحيح على localhost فقط.** يرتبط CDP بصرامة بـ `127.0.0.1:9333`. لا يمكن لأي مضيف خارجي على الشبكة المحلية أو شبكة المقهى أو شبكة مساحة العمل المشتركة الوصول إليه — مُتحقق عبر ربط socket، وليس عبر وعد.
- **صفر قياس عن بُعد.** نواة الموافقة التلقائية لا تُجري أي استدعاءات تحليلية أو تقارير أعطال أو "إحصائيات استخدام مجهولة". حركة المرور الصادرة الوحيدة هي إلى Telegram (فقط إذا قرنته) وإلى خادم التراخيص (فقط للتحقق من المفتاح).
- **إرفاق الأهداف بالقائمة البيضاء فقط.** طبقة CDP تُرفق **فقط** بأهداف `vscode-webview://` المطابقة للوحة Agent. ترفض الإرفاق بعلامات التبويب الأخرى أو وكلاء المتصفح الفرعيين أو أي هدف Chromium ليس من Antigravity.
- **حمولة نواة موقّعة.** يتم جلب محرك نواة Pro كحمولة موقّعة بـ Ed25519، ويتم التحقق منها داخل Node worker، وتُجمّع في RAM — لا تُكتب أبداً على القرص، ولا يمكن العبث بها عبر man-in-the-middle.
- **لا تعديل على Antigravity نفسه.** لا نُصلح ولا نُعيد تعبئة ولا نُعلّق (hook) على ثنائيات Antigravity. CDP خارجي بحت — قم بإلغاء التثبيت ولن يبقى أثر.

### 📱 2. التحكم عن بُعد عبر Telegram — قُد Antigravity من أي مكان

اقرن IDE الخاص بك مع بوت Telegram مرة واحدة، ويصبح هاتفك جهاز تحكم عن بُعد كاملاً لـ Agent:

- **`/prompt <نص>`** — أرسل prompt جديداً إلى المحادثة النشطة من أي مكان على الكوكب. يُدمج تلقائياً رسائل Telegram المقسّمة إلى 4096 حرفاً.
- **`/peek [N]`** — اقرأ أحدث رد لـ Agent مباشرة داخل Telegram، بدون الحاجة إلى إيقاظ حاسوبك.
- **أرسل صورة مع تعليق** — يُرفق لقطة الشاشة بالـ prompt للتصحيح المرئي أثناء التنقل.
- **`/pause` و `/resume`** — بدّل مفتاح الإيقاف العالمي من هاتفك. قفل الملف عبر النوافذ يضمن امتثال كل نافذة Antigravity فوراً.
- **`/stop`** — يُوقف الرد الذي يُولَّد حالياً.
- **أوامر موقّعة بـ HMAC.** يتم التحقق من كل أمر Telegram بمفتاح HMAC لكل جهاز. تسرّب محادثة Telegram *ليس* كافياً للتحكم في IDE الخاص بك.
- **انتخاب سيد واحد.** عندما تكون لديك 5 نوافذ Antigravity مفتوحة، نافذة واحدة فقط تفوز بقفل السيد وتُعالج الأوامر — لا إرسال مكرر، لا race conditions.

### 🛡️ 3. استراتيجية أول من نوعها في الصناعة لمقاومة الحظر

معظم أدوات النقر التلقائي تُعلَّم حسابات المستخدمين لأنها تضرب نقاط النهاية بإيقاع ثابت وتنقر بينما لا يزال الإنسان يكتب. Anti-Remote-Pro مُصمم بشكل معاكس تماماً:

- **توقيت بإيقاع إنساني.** polling نشط عند 500 مللي ثانية مع **±20% jitter عشوائي** + backoff أسّي (محدود بـ 30 ثانية) عند الأخطاء المتتالية. لا فواصل زمنية مثالية ميكانيكياً لأخذ البصمة.
- **سكون ذكي في الخمول.** بدون جلسة CDP نشطة، يسقط المحرك إلى heartbeat لكل 5 ثوانٍ — ضغط IPC قرب الصفر عندما تكون بعيداً.
- **نافذة سماح 20 ثانية للمستخدم النشط.** إذا كان المستخدم يكتب في الدردشة أو يُمرّر webview أو يُحرّك مؤشر المحرر، **تُؤجَّل جميع النقرات التلقائية**. البوت ينتظر حرفياً حتى تتوقف، فلا ينقر معك أبداً أثناء القراءة.
- **لا تصحيح ثنائيات.** ملفات Antigravity و settings.json ودليل بيانات المستخدم للقراءة فقط بالنسبة لهذا الامتداد. التحديثات التلقائية الرسمية وفحوصات السلامة تمر نظيفة لأنه لا يتم تعديل أي شيء.
- **CDP على loopback فقط.** لا مستمع TCP، لا توجيه عن بُعد. علامة `--remote-debugging-port=9333` هي علامة تصحيح Electron قياسية — نفس العلامة التي تستخدمها Microsoft و Google و JetBrains داخلياً.
- **جدار حماية أوامر صارم.** قائمة حظر مدمجة + قائمة سماح صارمة اختيارية + 50+ **Safety Presets** مُنتقاة (rm -rf, force-push, drop database, fork bombs, ماسحات الأقراص، حذف السجل…). البوت *سيرفض* تنفيذ أمر مدمر حتى لو أصر Agent.
- **نقرة واحدة لكل جلسة لأزرار التوسيع.** يُزيل حلقة "إعادة الفتح اللانهائية" التي قد تُشغّل استدلالات مكافحة إساءة الاستخدام.
- **تنحّي فوري للوكيل الفرعي الرسمي للمتصفح.** عندما تُفعّل أدوات المتصفح الخاصة بـ AG المنفذ 9222، يفصل worker الخاص بنا ويبتعد عن الطريق — لا تصادم منافذ، لا توقيع ارفاق مزدوج.

> الأثر المُركّب: من منظور backend لـ Agent، تبدو وكأنك مستخدم بشري أسرع قليلاً من المتوسط ولبق بلا عيب.

### 🔗 5. مزامنة متعددة الأجهزة مع تسجيل الدخول عبر Google — حساب واحد، كل آلاتك

**سجّل الدخول مرة واحدة بـ Google** ويتبعك Anti-Remote-Pro في كل مكان — سطح المكتب، الحاسوب المحمول، PC المنزل، محطة العمل، Windows و macOS و Linux.

- **Google OAuth بنقرة واحدة.** `Anti-Remote-Pro: Login with Google` يفتح المتصفح، يُكمل OAuth Google القياسي، ويُعيد التوجيه إلى IDE عبر معالج URI آمن `vscode://Pro.antigravity-autoaccept/auth`. بدون نسخ ولصق مفتاح الترخيص، بدون رقصة البريد الإلكتروني.
- **ترخيص Pro يُزامَن تلقائياً عبر الأجهزة.** اشتراكك مرتبط بحساب Google الخاص بك، وليس بآلة فردية. ثبّت على الجهاز رقم 2 و 3 و 10 — سجّل الدخول وينفتح Pro فوراً. خادم التراخيص يُتحقق من كل `machineId` مقابل حسابك عند كل تحديث.
- **تشغيل على عدد غير محدود من الأجهزة في نفس الوقت.** حاسوب العمل، سطح المكتب المنزلي، صندوق التطوير عن بُعد، VM لكل عميل — جميعها يمكنها تشغيل Anti-Remote-Pro بالتوازي تحت نفس الحساب.
- **أساطيل متوازية.** كل جهاز يعمل باستقلالية مع CDP / Swarm / جدار حماية أوامر خاص به، لذا يمكنك أن يكون لديك حاسوب محمول يعالج المحادثة A بينما يُشغّل سطح المكتب المحادثة B — لا أحد يسرق التركيز من الآخر.
- **اقتران Telegram لكل جهاز.** كل آلة تقترن مع بوت Telegram بشكل منفصل وتحصل على مفتاح HMAC خاص بها. يمكنك تسميتها في تدفق الاقتران (مثل "MacBook"، "WorkPC") ومخاطبتها بشكل فردي من نفس محادثة Telegram.
- **تنسيق عبر النوافذ على نفس الآلة.** عندما تكون لديك نوافذ Antigravity متعددة مفتوحة على جهاز واحد، يحافظ بروتوكول قفل الملف على مزامنة حالة pause/resume بينها في أقل من ثانية.
- **إلغاء فوري.** `Anti-Remote-Pro: Logout` يمسح الرمز المحلي ويُخبر الخادم بإبطال ذلك الجهاز. فقدت حاسوباً محمولاً؟ سجّل الخروج من آلة أخرى + `Refresh Pro Status` وتنطفئ الآلة القديمة.
- **Refresh Pro Status.** غيّرت الخطة أو جدّدت الاشتراك؟ أمر واحد يسحب أحدث الصلاحيات — بدون إعادة تثبيت، بدون إعادة تشغيل.

> في الاستخدام العملي: سجّل الدخول مرة على 3 حواسيب، اقرن كلاً منها بنفس بوت Telegram، ويصبح هاتفك جهاز تحكم مركزي لأسطول البرمجة بأكمله.

## ⚡ القدرات الأساسية لمحة سريعة

- **نقر تلقائي** Run / Accept / Always Allow / Allow this conversation / Continue / Retry — عبر لوحة Agent، مع وعي بالأولوية (`Run` يفوز دائماً على `Accept`).
- **الموافقة التلقائية على تعديلات الملفات** بمفتاح واحد — أوقفه في أي وقت لمراجعة diff يدوياً.
- **جدار حماية الأوامر** — وضع حظر الأنماط أو القائمة البيضاء الصارمة، مع مطابقة حدود الكلمة.
- **Safety presets** — استيراد بنقرة واحدة لـ 50+ نمط أمر مدمر.
- **وضع Swarm (Pro)** — يدور تلقائياً عبر محادثات Agent Manager المعلقة ويُشغّلها بدون تدخل في الخلفية.
- **Telegram Remote (Pro)** — تحكم IDE كامل: `/prompt`, `/peek`, prompts الصور, `/pause`, `/resume`, `/stop`, مع مصادقة HMAC.
- **تسجيل Google + مزامنة متعددة الأجهزة (Pro)** — حساب Google واحد، أجهزة غير محدودة.
- **إيقاف مؤقت عبر النوافذ** — الإيقاف المؤقت في نافذة Antigravity يُبث عبر قفل الملف إلى كل النوافذ الأخرى في < 1 ثانية.
- **سجل النشاط + لوحة التحكم** — شاهد كل زر تم النقر عليه، كل جلسة CDP، كل أمر Telegram مباشرة.
- **تفريغ التشخيص** — ينسخ حمولة تصحيح مُهيكلة إلى الحافظة للدعم برسالة واحدة.
- **اختصار Auto-Fix** — مُصحّح Windows `.lnk` لعلامة `--remote-debugging-port`.
- **الشفاء الذاتي بالـ heartbeat** — MutationObservers الميتة تُكتشف وتُعاد حقنها تلقائياً كل 10 ثوانٍ.

## ماذا يفعل

عندما يقترح Agent من Antigravity تعديلات ملفات أو أوامر طرفية أو يطلب أذونات أدوات، يُوافق هذا الامتداد عليها تلقائياً حتى لا تضطر إلى النقر على كل زر يدوياً.

**استراتيجيتان، بدون تداخل:**

| الاستراتيجية | ماذا تعالج | كيف |
|---|---|---|
| **VS Code Commands** (500 مللي ثانية + jitter) | خطوات Agent، أوامر الطرفية | يستدعي أوامر القبول الأصلية في Antigravity |
| **CDP + MutationObserver** (مدفوع بالأحداث) | Run, Accept, Always Allow, Continue, Retry | سكريبت مُحقن مرة واحدة، يستجيب فوراً لتغييرات DOM |

## الإعداد

### 1. تفعيل وضع التصحيح (مطلوب)

يحتاج الامتداد إلى Chrome DevTools Protocol للنقر على أزرار الأذونات. شغّل Antigravity بـ:

```
--remote-debugging-port=9333
```

> **لماذا المنفذ 9333؟** يستخدم Browser Control المدمج في Antigravity (زر Chrome في شريط الأدوات) المنفذ 9222 افتراضياً. استخدام نفس المنفذ يُسبب تصادم `EADDRINUSE` على macOS/Linux. المنفذ 9333 يتجنب ذلك تماماً.

### 2. تثبيت الامتداد

**من VSIX (موصى به):**
1. احصل على أحدث حزمة `.vsix`
2. في Antigravity: `Ctrl+Shift+P` → `Extensions: Install from VSIX`

## سير عمل Multi-Agent

### ⚠️ قيد Agent Manager (بدون وضع Swarm)

يستخدم Agent Manager في Antigravity **webview واحداً مشتركاً** — فقط DOM للمحادثة النشطة يُعرض. المحادثات الخلفية تكون unmounted تماماً. بدون وضع Swarm، يمكن لـ VS Code Commands API و CDP الوصول فقط إلى المحادثة المرئية حالياً.

### ✅ الحل: وضع Swarm (Pro)

يحل وضع Swarm هذا بالتناوب على التركيز عبر جميع المحادثات المعلقة، مما يسمح لعشرات الوكلاء الخلفيين بالعمل بدون تدخل:

- يمسح كل محادثة في الشريط الجانبي ويختار التي لديها عمل معلق.
- يحترم إعداد idle-seconds الخاص بك حتى لا يسرق التركيز أثناء الكتابة.
- واعٍ بالنوافذ المتعددة: إيقاف أي نافذة يُوقف الجميع عبر بث قفل الملف.
- آمن افتراضياً: حمولة نواة موقّعة، جدار حماية الأوامر لا يزال يُطبّق.

### بديل: مساحة عمل مكررة (مجانية)

إذا لم يكن لديك ترخيص Pro، يمكنك تشغيل 2-3 وكلاء بالتوازي:

1. انقر على **File → Duplicate Workspace**
2. يفتح نافذة Antigravity ثانية متصلة بنفس المشروع
3. ابدأ دردشة في النافذة 1 ودردشة أخرى في النافذة 2
4. كل نافذة لها webview خاص بها — الامتداد ينقر تلقائياً على الأزرار في **كلتا النافذتين في نفس الوقت**

## كيف يعمل

### CDP دائم + MutationObserver

يحتفظ الامتداد بـ **اتصال WebSocket دائم على مستوى المتصفح** إلى DevTools Protocol في Chromium. بدلاً من polling كل 1.5 ثانية، يُحقن حمولة **MutationObserver** مرة واحدة لكل هدف. يستجيب observer فوراً عندما يُركّب React عناصر أزرار جديدة، مع throttle leading-edge 100 مللي ثانية لمنع ذروات CPU أثناء المخرجات المتدفقة. يستخدم الامتداد **فلتر أهداف قائمة بيضاء فقط**.

### ماسح DOM بتمرير واحد

يجتاز ماسح الأزرار شجرة DOM **مرة واحدة بالضبط** لكل دورة، فاحصاً جميع الكلمات المفتاحية في نفس الوقت مقابل كل عقدة. هذا O(D) بدلاً من O(N×D) الذي قد يُسبب تجميد واجهة المستخدم مع العديد من الكلمات المفتاحية. المطابقة الواعية بالأولوية تضمن أن `Run` يفوز دائماً على `Accept`، الذي يفوز دائماً على `Allow`.

### حارس Webview

تعمل لوحة Agent في Antigravity في عملية Chromium معزولة (OOPIF). يستخدم السكريبت المُحقن فحص `isAgentPanel()` مؤجل داخل `scanAndClick()` — يتحقق من وجود `.react-app-container` ديناميكياً في كل مسح.

### الشفاء الذاتي بالـ Heartbeat

يُتحقق اتصال CDP من الجلسات الموجودة في كل دورة heartbeat (10 ثوانٍ). إذا كان MutationObserver لجلسة ميتاً، فإنه يُعيد حقن observer تلقائياً — بدون الحاجة إلى إعادة اتصال. الجلسات غير القابلة للوصول 3 مرات متتالية تُفصل وتُقطع بشكل نظيف.

### منع حلقة زر التوسيع

تستخدم أزرار نوع التوسيع قاعدة **النقر-مرة-واحدة-لكل-جلسة**: بمجرد النقر عليها، يتم قمعها بشكل دائم لتلك جلسة CDP. هذا يمنع حلقة إعادة فتح overlay اللانهائية.

### اكتشاف الأزرار

داخل لوحة Agent، يبحث `TreeWalker` عن الأزرار حسب المحتوى النصي باستخدام مطابقة `startsWith` مع **فحوصات حدود الكلمات** لمنع الإيجابيات الخاطئة:

| الأولوية | النص | يطابق |
|---|---|---|
| 1 | `run` | زر "Run Alt+d" |
| 2 | `accept` | زر Accept |
| 3 | `always allow` | مطالبات الأذونات |
| 4 | `allow this conversation` | أذونات نطاق المحادثة |
| 5 | `allow` | مطالبات الأذونات |
| 6 | `retry` | مطالبات Retry |
| 7 | `continue` | استئناف بعد حد استدعاء Agent |

### تصفية الأوامر

تستخدم قوائم الأوامر المحظورة والمسموح بها **مطابقة حدود الكلمات** مقابل كتلة الكود فوق زر Run. على سبيل المثال، حظر `rm` سيحظر `rm -rf /tmp` لكن **ليس** `yarn format` أو `npm run build`.

## استكشاف الأخطاء

### يتوقف البوت عن العمل بعد بضع ساعات

**السبب:** إما (a) يُعيد Antigravity تشغيل عملية Electron بصمت والعملية الجديدة لا تحتوي على `--remote-debugging-port=9333`، أو (b) تم مسح سياق تنفيذ webview بواسطة تنقل / hot-reload.

**الإصلاح:** منطق الشفاء الذاتي بالـ heartbeat يكتشف ويُعيد حقن المراقبين الميتين تلقائياً. إذا لم يعمل، أغلق **جميع** نوافذ Antigravity تماماً، ثم افتحها من اختصارك المُصحّح.

### البوت ON لكن لا ينقر على أي شيء

1. **بدّل OFF → ON** — انقر على أيقونة شريط الحالة مرتين لإعادة بدء polling
2. **تحقق من منفذ التصحيح** — قم بزيارة `http://127.0.0.1:9333/json/list` في متصفح. إذا رفض، فمنفذ التصحيح ميت
3. **تحقق من سجلات Output** — `Ctrl+Shift+U` → القائمة المنسدلة → `Anti-Remote-Pro`. ابحث عن أسطر `[CDP] ✓ Thread`

### لا تظهر أيقونة شريط الحالة بعد التثبيت

1. شغّل `Ctrl+Shift+P` → `Reload Window`
2. تحقق من أن VSIX بُني **مع** التبعيات (يجب أن تكون حزمة `ws` مُضمّنة)

## أسئلة أمان شائعة

**لماذا يحتاج هذا إلى `--remote-debugging-port`؟**

تعمل لوحة Agent في Antigravity في عملية Chromium معزولة. لا يمكن لـ VS Code Extension API رؤية أو التفاعل مع أزرار Run / Accept / Allow بداخلها — فهي عناصر واجهة React بدون أوامر مُسجّلة. Chrome DevTools Protocol (CDP) على منفذ localhost (افتراضياً 9333) هو الطريقة الوحيدة للوصول إليها.

**هل هو آمن؟**

- **localhost فقط** — يرتبط المنفذ بـ `127.0.0.1`، وليس `0.0.0.0`. لا يمكن لأي آلة خارجية الاتصال.
- **صفر قياس عن بُعد في نواة الموافقة التلقائية** — مسح DOM ومطابقة الأزرار وتصفية الأوامر لا تُجري أبداً استدعاءات شبكة.
- **حركة المرور الصادرة الوحيدة** هي (a) التحقق من الترخيص، (b) Telegram Bot API — فقط إذا قرنته، و(c) جلب حمولة النواة الموقّعة لوضع Swarm. الثلاثة متوقفة افتراضياً.
- **حمولة Pro موقّعة.** محرك نواة Swarm موقّع بـ Ed25519 ومُتحقق منه داخل Node worker قبل التجميع في RAM.
- **سير عمل تطوير قياسي** — `--remote-debugging-port` هي نفس العلامة التي يستخدمها مطورو امتدادات VS Code.
- **مُصحّح الاختصارات محدود** — يُعدّل الإصلاح التلقائي فقط ملفات `.lnk` التي يحتوي مسارها الهدف على "Antigravity".
- **لا يتم تعديل أي ملف Antigravity** — يعمل هذا الامتداد بالكامل كامتداد VS Code خارجي.

**هل ستقرأ محادثة Telegram الخاصة بي كودي؟**

فقط ما تطلبه *أنت* صراحةً. `/peek` يُرجع النص المرئي للوحة Agent النشطة؛ `/prompt` يُرسل رسالتك إلى الداخل. الامتداد لا يدفع أبداً بشكل استباقي الكود أو diffs أو لقطات الشاشة أو القياس عن بُعد إلى Telegram.
