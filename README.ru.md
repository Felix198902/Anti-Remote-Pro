## 🔥 Почему Anti-Remote-Pro

### 🔒 1. Абсолютная приватность — ваш код никогда не покидает ноутбук

- **100 % локальный конвейер.** Всё определение кнопок, сканирование DOM и фильтрация команд работают внутри вашего процесса VS Code / Antigravity. Ни один промпт, скриншот или diff никогда не покидает машину.
- **Порт отладки только на localhost.** CDP жёстко привязан к `127.0.0.1:9333`. Ни один внешний хост в LAN, в Wi-Fi кафе или коворкинга не может его достичь — гарантировано не обещанием, а bind'ом сокета.
- **Нулевая телеметрия.** Ядро авто-принятия не делает аналитических вызовов, отчётов о падениях, «анонимной статистики использования». Единственный исходящий трафик — к Telegram (только если вы его сопрягли) и к серверу лицензий (только для проверки ключа).
- **Подключение к target только по белому списку.** Слой CDP подключается **только** к target'ам `vscode-webview://`, соответствующим панели Agent. Он отказывается подключаться к другим вашим вкладкам, под-агентам браузера или любому Chromium-target'у не от Antigravity.
- **Подписанный полезный груз ядра.** Ядро Pro загружается как payload с подписью Ed25519, проверяется внутри Node worker и компилируется в RAM — никогда не пишется на диск, не подделывается man-in-the-middle.
- **Никаких модификаций самого Antigravity.** Мы не патчим, не переупаковываем и не хукаем бинарники Antigravity. Чисто внешний CDP — удалите расширение и не останется никаких следов.

### 📱 2. Удалённое управление через Telegram — управляйте Antigravity откуда угодно

Один раз сопрягите вашу IDE с Telegram-ботом, и ваш телефон станет полноценным пультом для Agent:

- **`/prompt <текст>`** — отправьте новый промпт в активный разговор из любой точки планеты. Автоматически склеивает разделённые сообщения по 4096 символов от Telegram.
- **`/peek [N]`** — читайте свежий ответ Agent прямо в Telegram, не нужно будить ноутбук.
- **Отправка фото с подписью** — прикрепляет скриншот к промпту для визуальной отладки на ходу.
- **`/pause` и `/resume`** — переключайте глобальный kill-switch с телефона. Кросс-оконный файловый замок гарантирует, что каждое окно Antigravity подчиняется мгновенно.
- **`/stop`** — останавливает текущий генерируемый ответ.
- **Команды с HMAC-подписью.** Каждая команда Telegram проверяется по уникальному для устройства HMAC-ключу. Утечка чата Telegram *недостаточна* для управления вашей IDE.
- **Выбор единственного master'а.** Когда у вас открыто 5 окон Antigravity, только одно выигрывает master-замок и обрабатывает команды — никаких дублей отправки, никаких race conditions.

### 🛡️ 3. Первая в индустрии стратегия защиты от блокировки

Большинство авто-кликеров получают пометку аккаунта, потому что долбят эндпойнты с фиксированной частотой и кликают, пока человек ещё печатает. Anti-Remote-Pro спроектирован противоположно:

- **Человеческий ритм таймингов.** Активный поллинг 500 мс с **±20 % случайным джиттером** + экспоненциальный backoff (с потолком 30 с) при последовательных ошибках. Никаких машинно-идеальных интервалов для фингерпринта.
- **Smart-sleep при простое.** Без активной CDP-сессии движок падает до heartbeat'а раз в 5 с — практически нулевая нагрузка IPC, когда вас нет.
- **20-секундное окно деликатности при активном пользователе.** Если пользователь печатает в чате, прокручивает webview или двигает курсор редактора, **все авто-клики откладываются**. Бот буквально ждёт, пока вы не остановитесь, поэтому никогда не кликает рядом, пока вы читаете.
- **Никаких бинарных патчей.** Файлы Antigravity, settings.json и каталог пользовательских данных доступны этому расширению только для чтения. Официальные авто-обновления и проверки целостности проходят чисто, потому что ничего не меняется.
- **CDP только loopback.** Никакого TCP listener'а, никакой удалённой переадресации. Флаг `--remote-debugging-port=9333` — это обычный отладочный флаг Electron'а — тот же, что используют внутри Microsoft, Google и JetBrains.
- **Жёсткий командный firewall.** Встроенный blocklist + опциональный строгий allowlist + 50+ кураторских **Safety Presets** (rm -rf, force-push, drop database, fork-бомбы, очистители диска, удаления реестра…). Бот *откажется* выполнить деструктивную команду, даже если Agent настаивает.
- **Click-once-per-session для кнопок expand.** Устраняет цикл «бесконечного переоткрытия», который рискует сработать на эвристиках anti-abuse.
- **Мгновенная уступка официальному под-агенту Browser.** Когда собственная браузерная утилита AG активирует порт 9222, наш worker отсоединяется и уходит с дороги — никаких коллизий портов, никакой подписи двойного attach'а.

> Совокупный эффект: с точки зрения backend'а Agent вы выглядите как чуть более быстрый, чем средний, безупречно вежливый человек-пользователь.

## ⚡ Ключевые возможности кратко

- **Авто-клик** Run / Accept / Always Allow / Allow this conversation / Continue / Retry по всей панели Agent — с приоритетом, так что `Run` всегда побеждает `Accept`.
- **Авто-принятие правок файлов** одним переключателем — выключите в любой момент, чтобы вручную просматривать diff'ы.
- **Командный firewall** — режим блокировки по шаблону или строгий whitelist, оба с матчингом по границе слова.
- **Safety presets** — импорт в один клик 50+ деструктивных шаблонов команд (очистители файловых систем, форматтеры дисков, force-push, drop'ы БД, fork-бомбы).
- **Режим Swarm (Pro)** — автоматически переключается между ожидающими разговорами Agent Manager и выполняет их без участия рук в фоне.
- **Telegram Remote (Pro)** — полный удалённый IDE: `/prompt`, `/peek`, фото-промпты, `/pause`, `/resume`, `/stop`, с HMAC-аутентификацией.
- **Вход через Google + мульти-устройственная синхронизация (Pro)** — один аккаунт Google, неограниченное число устройств; OAuth-callback через `vscode://`-URI-обработчик; верификация лицензии для каждого устройства.
- **Кросс-оконная пауза** — пауза в одном окне Antigravity транслируется через файловый замок во все остальные окна за <1 с.
- **Лог активности + Дашборд** — смотрите каждую нажатую кнопку, каждую CDP-сессию, каждую команду Telegram в реальном времени.
- **Диагностический дамп** — копирует структурированный отладочный payload в буфер обмена для саппорта одним сообщением.
- **Auto-Fix shortcut** — патчер `.lnk` для Windows (Рабочий стол / Меню Пуск / Панель задач) для флага `--remote-debugging-port`.
- **Самовосстановление по heartbeat** — мёртвые MutationObserver'ы (стёртые навигацией webview / hot-reload) обнаруживаются и переинъектируются автоматически каждые 10 с.

## Что он делает

Когда Agent Antigravity предлагает правки файлов, команды терминала или просит разрешения для инструментов, это расширение автоматически принимает их, чтобы вам не пришлось кликать по каждой кнопке вручную.

**Две стратегии, ноль помех:**

| Стратегия | Что обрабатывает | Как |
|---|---|---|
| **Команды VS Code** (500 мс + джиттер) | Шаги Agent, команды терминала | Вызывает нативные команды принятия Antigravity |
| **CDP + MutationObserver** (событийная) | Run, Accept, Always Allow, Continue, Retry | Скрипт инъектируется один раз, мгновенно реагирует на изменения DOM |

## Установка

### 1. Включить режим отладки (Обязательно)

Расширению нужен Chrome DevTools Protocol для нажатия кнопок разрешений. Запускайте Antigravity с:

```
--remote-debugging-port=9333
```

> **Почему порт 9333?** Встроенный Browser Control Antigravity (кнопка Chrome в панели инструментов) по умолчанию использует порт 9222. Использование того же порта вызывает конфликт `EADDRINUSE` на macOS/Linux. Порт 9333 полностью обходит это.

<summary><b>🪟 Windows</b></summary>

**Автоматически:** При первом запуске расширение определяет, что порт закрыт, и показывает **«Auto-Fix Shortcut»** — щёлкните, чтобы автоматически пропатчить ваш `.lnk` ярлык.

**Вручную:** Правый клик по ярлыку Antigravity → Свойства → добавить к Объекту:
```
--remote-debugging-port=9333
```

</details>

<details>
<summary><b>🍎 macOS</b></summary>

**Вариант 1 — приложение Automator (рекомендуется):**

2. Найдите **«Запустить shell-скрипт»** в библиотеке
3. Вставьте: `open -a "Antigravity" --args --remote-debugging-port=9333`
4. Сохраните как «AntiGravity Launcher» на Рабочий стол или в Программы

**Вариант 2 — alias терминала** (добавить в `~/.zshrc`):

> **Заметка:** Имя программы должно совпадать точно. Проверьте с помощью `ls /Applications/ | grep -i anti`

**Вариант 3 — прямой бинарник Electron** (если `open -a` неправильно передаёт аргументы):
```bash
alias antigravity='/Applications/Antigravity.app/Contents/MacOS/Electron --remote-debugging-port=9333 & disown'
```

</details>

```bash

```

**Вариант 3 — wrapper-скрипт:**

### 2. Установить расширение

**Из VSIX (рекомендуется):**
1. Получите последний `.vsix`-пакет
2. В Antigravity: `Ctrl+Shift+P` → `Extensions: Install from VSIX`
3. Выберите загруженный файл
4. Reload Window

**Вручную:**
1. Скопируйте директорию `src/`, `package.json` и `package-lock.json` в:
   ```
   ~/.antigravity/extensions/Pro.antigravity-autoaccept-<version>/
   ```
2. Запустите `npm install` в этой директории (устанавливает зависимость `ws`)
3. Reload Window

## Использование

### Базовые команды

- **Переключение:** Щёлкните по `⚡ Auto: ON` / `✕ Auto: OFF` в строке состояния
- **Или:** `Ctrl+Shift+P` → `Anti-Remote-Pro: Toggle ON/OFF`
- **Дашборд:** Щёлкните по иконке дашборда в строке состояния, чтобы увидеть статус CDP, активные сессии, лог активности и выбор языка

## Multi-Agent Workflow

### ⚠️ Ограничение Agent Manager (без режима Swarm)

Agent Manager Antigravity использует **единый общий webview** — рендерится только DOM активного разговора. Фоновые разговоры полностью unmounted. Без режима Swarm и API VS Code Commands, и CDP могут достичь только текущего видимого разговора.

### ✅ Решение: режим Swarm (Pro)

Режим Swarm решает это, прокручивая фокус по всем ожидающим разговорам, позволяя десяткам фоновых агентов работать без вмешательства:

- Сканирует каждый разговор в боковой панели и выбирает тот, у которого есть ожидающая работа.
- Уважает вашу настройку idle-seconds, чтобы никогда не красть фокус, пока вы печатаете.
- Учитывает кросс-оконность: пауза в любом окне ставит на паузу все через broadcast по файловому замку.
- Безопасно по умолчанию: подписанный payload ядра, командный firewall всё ещё применяется.

### Запасной вариант: Дублирование Workspace (бесплатно)

Если у вас нет Pro-лицензии, вы всё ещё можете запустить 2–3 агента параллельно:

1. Щёлкните **File → Duplicate Workspace**
2. Это откроет второе окно Antigravity, подключённое к тому же проекту
3. Начните чат в Окне 1 и другой чат в Окне 2
4. Каждое окно имеет свой webview — расширение автокликает кнопки в **обоих окнах одновременно**

## Как это работает

### Постоянный CDP + MutationObserver

Расширение поддерживает **постоянное браузерного уровня WebSocket-соединение** к DevTools Protocol Chromium. Вместо поллинга каждые 1,5 с оно инъектирует payload **MutationObserver** один раз на target. Observer мгновенно реагирует, когда React монтирует новые элементы кнопок, с throttle leading-edge 100 мс для предотвращения скачков CPU во время потокового вывода. Расширение использует **фильтр target'ов только по whitelist** — оно подключается только к target'ам `vscode-webview://` и автоматически уступает CDP-порт при обнаружении под-агента браузера AG, предотвращая конфликты ArrayBuffer.

### Single-Pass DOM-сканер

Сканер кнопок проходит по DOM-дереву **ровно один раз** за цикл, проверяя все ключевые слова одновременно против каждого узла. Это O(D) вместо O(N×D), что могло бы вызывать заморозки UI при большом числе ключевых слов. Матчинг с учётом приоритетов гарантирует, что `Run` всегда побеждает `Accept`, который всегда побеждает `Allow`, независимо от порядка DOM.

### Webview Guard

Панель Agent Antigravity работает в изолированном Chromium-процессе (OOPIF). Инъектированный скрипт использует отложенную проверку `isAgentPanel()` внутри `scanAndClick()` — динамически проверяя существование `.react-app-container` при каждом скане, а не во время инъекции. Это избегает race condition'а, когда DOM не гидратирован на `targetCreated`.

### Самовосстановление по heartbeat

CDP-соединение валидирует существующие сессии на каждом цикле heartbeat (10 с). Если MutationObserver сессии мёртв (контекст выполнения очищен навигацией webview или hot-reload'ом React), она автоматически переинъектирует observer — без необходимости переподключения. Сессии, недоступные 3 раза подряд, чисто отсоединяются и удаляются.

### Предотвращение цикла кнопки expand

Кнопки типа expand (например, «Expand» в превью браузера) используют правило **click-once-per-session**: после нажатия они навсегда подавляются для этой CDP-сессии через Set `expandedOnce`. Это предотвращает бесконечный цикл переоткрытия overlay, где закрытие развёрнутой панели триггерит повторный клик. Состояние естественно сбрасывается, когда начинается новый разговор Agent.

### Обнаружение кнопок

Внутри панели Agent `TreeWalker` ищет кнопки по текстовому содержимому, используя `startsWith`-матчинг с **проверками границы слова**, чтобы предотвратить ложные срабатывания (например, `accept-test.js` не совпадёт с `accept`):

| Приоритет | Текст | Совпадает с |
|---|---|---|
| 1 | `run` | кнопка «Run Alt+d» (не выпадающее «Always run ^») |
| 2 | `accept` | кнопка Accept |
| 3 | `always allow` | промпты разрешений |
| 4 | `allow this conversation` | разрешения области разговора |
| 5 | `allow` | промпты разрешений |
| 6 | `retry` | промпты Retry |
| 7 | `continue` | возобновление после лимита вызовов Agent |

### Фильтрация команд

Списки заблокированных и разрешённых команд используют **матчинг по границе слова** против блока кода над кнопкой Run. Например, блокировка `rm` заблокирует `rm -rf /tmp`, но НЕ `yarn format` или `npm run build`.

## Устранение неполадок

### Бот перестаёт работать через несколько часов

**Причина:** Либо (a) Antigravity молча перезапускает свой процесс Electron (авто-обновления, давление памяти или crash extension host'а) и новый процесс не имеет `--remote-debugging-port=9333`, либо (b) контекст выполнения webview был очищен навигацией / hot-reload'ом.

**Исправление:** Логика самовосстановления по heartbeat автоматически обнаруживает и переинъектирует мёртвые observer'ы. Если всё равно не работает, **полностью** закройте все окна Antigravity, затем заново откройте из вашего пропатченного ярлыка.

### Бот ON, но ничего не кликает

1. **Переключите OFF → ON** — щёлкните по иконке в строке состояния дважды, чтобы перезапустить поллинг
2. **Проверьте порт отладки** — посетите `http://127.0.0.1:9333/json/list` в браузере. Если он отказывает, порт отладки мёртв (см. выше)
3. **Проверьте логи Output** — `Ctrl+Shift+U` → выпадающий список → `Anti-Remote-Pro`. Ищите строки `[CDP] ✓ Thread`. Если их нет, CDP не может найти панель Agent

### Иконка строки состояния не появляется после установки

1. Запустите `Ctrl+Shift+P` → `Reload Window`
2. Проверьте, что VSIX был собран **с** зависимостями (пакет `ws` должен быть включён)

## FAQ по безопасности

**Зачем нужен `--remote-debugging-port`?**

Панель Agent Antigravity работает в изолированном Chromium-процессе. API расширений VS Code не может видеть или взаимодействовать с кнопками Run / Accept / Allow внутри неё — это React-UI-элементы без зарегистрированных команд. Chrome DevTools Protocol (CDP) на localhost-порте (по умолчанию 9333) — единственный способ их достичь.

**Это безопасно?**

- **Только localhost** — порт привязывается к `127.0.0.1`, а не к `0.0.0.0`. Никакая внешняя машина не может подключиться.
- **Нулевая телеметрия в ядре авто-принятия** — сканирование DOM, матчинг кнопок и фильтрация команд никогда не делают сетевых вызовов.
- **Единственный исходящий трафик** — это (a) проверка лицензии, (b) Telegram Bot API — и только если вы его сопрягли, и (c) загрузка подписанного payload'а ядра для режима Swarm. Все три выключены по умолчанию.
- **Подписанный Pro payload.** Ядро Swarm подписано Ed25519 и проверяется внутри Node worker'а до того, как скомпилируется в RAM — man-in-the-middle не может подсунуть зловредный сканер.
- **Стандартный dev-workflow** — `--remote-debugging-port` — это тот же флаг, который используется разработчиками расширений VS Code и для отладки Electron-приложений.
- **Патчер ярлыков ограничен по области** — авто-фикс модифицирует только `.lnk`-файлы, целевой путь которых содержит «Antigravity».
- **Никаких файлов Antigravity не модифицируется** — это расширение работает полностью как внешнее расширение VS Code. Удалите и любой след исчезнет.

**Будет ли мой Telegram-чат читать мой код?**

Только то, что *вы* явно запросите. `/peek` возвращает видимый текст активной панели Agent; `/prompt` отправляет ваше сообщение туда. Расширение никогда проактивно не пушит код, diff'ы, скриншоты или телеметрию в Telegram.
